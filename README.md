# SQL Data Analysis

## Project Overview

This project was completed as part of the DecodeLabs Data Analytics Internship. The objective of this project was to perform SQL based data analysis on a cleaned dataset by creating a database, executing SQL queries, and generating meaningful business insights. SQLite was used as the database management system and Python was used to connect, manage, and analyze the data.

## Objective

The main objectives of this project are:

- Create a database from a cleaned dataset
- Import data into an SQL table
- Perform data analysis using SQL queries
- Generate business insights from the dataset
- Apply SQL concepts such as filtering, sorting, grouping, and aggregation

## Tools and Technologies Used

- Python
- Pandas
- SQLite
- Google Colab
- Jupyter Notebook

## Dataset

The cleaned dataset generated during Project 1 was used for SQL based analysis. The dataset was imported into an SQLite database and stored in a table for querying and analysis.

## Project Workflow

### 1. Data Import

- Loaded the cleaned dataset using Pandas
- Connected to an SQLite database
- Imported the dataset into an SQL table

### 2. Database Creation

- Created a local SQLite database
- Stored records in the orders table

### 3. SQL Query Execution

Executed various SQL queries to analyze the dataset and generate insights.

## SQL Concepts Implemented

### SELECT

Retrieved records from the database table.

### WHERE

Filtered records based on specific conditions.

### ORDER BY

Sorted records in ascending and descending order.

### COUNT

Calculated the total number of records.

### AVG

Calculated average values from numerical columns.

### SUM

Calculated total sales and revenue values.

### GROUP BY

Grouped data based on product categories and other attributes.

### HAVING

Filtered grouped data using aggregate conditions.

### Aliases AS

Used aliases to provide meaningful names for calculated columns.

## Analysis Performed

### Total Orders Analysis

Calculated the total number of orders present in the dataset.

### Sales Analysis

Calculated total sales and average sales values.

### Product Performance Analysis

Identified products with the highest number of orders.

### Revenue Analysis

Calculated revenue generated by each product.

### Order Status Analysis

Analyzed order status and delivery performance.

## Key Findings

- Total orders were successfully calculated using SQL aggregation functions.
- Revenue and sales trends were identified using SUM and AVG functions.
- Product level performance was analyzed using GROUP BY queries.
- SQL filtering techniques helped identify important business records.
- Aggregation queries generated meaningful business insights from the dataset.
